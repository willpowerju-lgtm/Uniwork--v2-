# Backlink Schema · 反向链接规范

> **唯一判定标准：只有"显式、可解析的 `[[引用]]` 关系"才产生反向链接。**
> 任何反向链接实现都必须实现本规范、且每条产出都要过 §6 校验。代码里的唯一实现是 `index.html` 的 `BL` 模块（`resolveNote` / `backlinks` / 编辑器双链着色全部走它）。

---

## 1. 术语

- **Note（笔记）**：Vault 内的一个 `.md` 文件。**只有 `.md` 参与反向链接**——docx / xlsx / pdf 不解析、既不作来源也不作目标。
- **Wikilink（双链）**：来源笔记正文里形如 `[[目标]]` 或 `[[目标|别名]]` 的标记。
- **Source（来源 S）**：正文里含有 wikilink 的笔记。
- **Target（目标 T）**：某个 wikilink 解析到的笔记。
- **反向链接（backlink of T）**：所有"显式 `[[…]]` 解析到 T"的来源 S。

## 2. Wikilink 语法

```
[[  TARGET  ( | ALIAS )?  ]]
```
- `TARGET`：**非空**，不含 `[`、`]`、`|`。
- `ALIAS`：可选显示别名，**不参与解析**（只影响显示文字 / 高亮文字）。
- 正则（唯一）：`/\[\[([^\[\]|]+?)(?:\|([^\[\]]*))?\]\]/g`
- 纯文本里出现笔记名、或名字作为子串出现，**都不算引用**。只有 `[[…]]` 标记才算。

## 3. 目标归一化 `norm(target)`

依次：① 去首尾空白 ② 去结尾扩展名 `.md|.markdown|.txt|.docx` ③ Unicode `NFC` ④ 转小写（大小写不敏感）。
（保留中间的 `/`，用于路径限定 `[[文件夹/笔记]]`。）

## 4. 解析 `resolve(target, notes) → Note | null`

`notes` 先过滤为只剩 `.md` 笔记。

- **路径限定**（`target` 含 `/`）：候选 = `norm(笔记 path 去扩展名)` 以 `norm(target)` 结尾的笔记；按优先级取第一个；无 → `null`。
- **按名解析**（不含 `/`）：候选 = `canonical(笔记)` === `norm(target)` 的笔记（`canonical` = 文件名去扩展名后 `norm`）。
  - 1 个 → 返回它
  - 多个（**重名**）→ 按**确定性优先级**取第一个：先 `WIKI/` 下的，再按 `path` 字典序
  - 0 个 → `null`（**未解析链接，不产生反向链接**）

> 解析返回的是"**具体的某个笔记对象**"。判等一律用 `note.id`，不是用名字——重名笔记不会互相串台。

## 5. 反向链接判定 `isBacklink(S, T)`

S 是 T 的反向链接 **当且仅当**同时满足：

1. S、T 都是 `.md` 笔记（§1）
2. `S.id ≠ T.id`（**禁止自引用**）
3. S 的**磁盘 markdown 原文** `S.curMd`（未加载 = `null` 时**跳过该来源**，绝不拿渲染后的 HTML / 名字去猜）中，**存在至少一个 wikilink `w`**，使得 `resolve(w.target).id === T.id`（解析到**同一个文件**，不是仅"同名"）

## 6. 校验 `validate(S, T, w)`（产出前必过）

每一条要展示的反向链接，落地前都要过：
`isNote(S) && isNote(T) && S.id≠T.id && w.target 非空 && resolve(w.target).id === T.id`。
不过 → 开发期 `console.warn('[backlink schema] 违规丢弃 …')`，且**不计入**。

## 7. 不变量（任何 backlink 实现都必须满足）

| # | 不变量 |
|---|--------|
| I1 | 无自引用（S≠T） |
| I2 | "未解析链接"（resolve→null）不产生反向链接 |
| I3 | 仅显式 `[[…]]` 触发——纯文本提名字、子串包含都不算 |
| I4 | 判等按 `id`，重名不串台 |
| I5 | 来源只取 `.md` 原文（`curMd`），不拿 HTML/猜测 |
| I6 | 解析规则与"编辑器里双链着色（蓝=已解析/红=未解析）、点击跳转"**完全一致**（共用 `BL.resolve`） |

## 8. 自检 `BL.audit(notes)`

开发期可在 console 跑 `BL.audit()`：遍历所有笔记对，列出每条反向链接及其命中的 `[[…]]`，并对每条跑 §6 校验，返回 `{ pairs, violations }`。`violations` 非空即说明实现违反了本规范。
